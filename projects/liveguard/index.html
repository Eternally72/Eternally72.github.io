项目 02 / 智能运维

LiveGuard

面向实时音视频业务的 AI 体验巡检助手，将 CDN、直播云、RTC 和边缘节点等底层指标，转化为可理解、可验证和可执行的根因判断、处理建议与复盘报告。

  • React
  • TypeScript
  • FastAPI
  • Pydantic
  • LangChain
  • LangGraph
  • Ant Design
  • ECharts

项目概览

直播发生卡顿、延迟或黑屏时，告警通常只能说明“出现异常”，却无法直接回答问题来自主播网络、CDN 缓存、区域边缘节点还是源站服务。LiveGuard 在基础监控之上增加体验解释与故障响应能力，帮助直播运营、客户成功和技术支持完成异常发现、根因定位、处置沟通与事件复盘。

当前能力边界

当前业务监控指标和故障场景均由 Mock 数据生成；AI 诊断可以真实调用百炼 LLM。系统会在调用模型前移除注入故障类型和预设答案，要求 LLM 仅根据数值快照、趋势与地域分布独立归因。

核心流程

  • 聚合延迟、卡顿率、丢包率、首帧时间、CDN 命中率、回源请求及 4xx/5xx 等观测指标。
  • 规则引擎识别异常事实，并生成只包含白名单观测数据的盲诊断输入。
  • LLM 输出故障类别、关键证据、最可能根因、其他可能性和验证步骤。
  • 服务端计算证据匹配度；结论缺乏数据支撑时拒绝结果并切换到规则诊断。
  • 输出处理建议、客户沟通话术和结构化事件复盘报告。

可靠性与安全机制

  • 诊断输入采用字段白名单，故障标签与预设答案不会进入模型上下文。
  • 使用 LangGraph 编排数据清洗、模型调用、结构化解析与结果构造。
  • 对模型输出执行结构校验和服务端证据评分，不采用模型自报置信度。
  • 模型未配置、超时、格式异常或证据不足时自动使用规则诊断。
  • 页面明确展示诊断来源，避免把降级结果伪装成模型结论。

覆盖场景

当前覆盖电商直播、在线课堂、企业会议、视频客服和赛事直播五类业务场景，并可稳定复现主播弱网、区域链路异常、CDN 命中率下降、源站 5xx、热点资源未预热和推流中断六类典型异常。
